람다식 : 익명 구현 객체를 식으로 표현

람다식으로 사용할 수 있는 타켓 타입(인터페이스)
 - 추상 메서드를 하나만 가지고 있어야 함
 - @FunctionalInterface 컴파일 시 오류 검출 => 추상 메서드가 하나인지 검증

 표준 API의 함수적 인터페이스
 1) Consumer : 매개값은 있고 o, 리턴값 없음  (소비의 의미)
    void accept(T t) : 받아서 소비

 2) Supplier : 매개값은 없고, 리턴값은 있음 o  (공급의 의미)
    T get()  : 공급(반환)... 객체로 무조건

3) Function : 매개값도 있고, 리턴값도 있음 / 주로 매개값을 리턴값으로 매핑
 4) Operator : 매개값도 있고 o, 리턴값도 있음o / 주로 매개값을 연산하고 결과를 리턴 
 5) Preidcate : 매개값은 있고 o, 리턴값은 boolean / 매개값을 조사하여 t/f 리턴

  - 메서드 참조
   하나의 메서드만 호출하는 람다식은 아래와 같이 사용
   클래스 이름::메서드 이름::참조변수::메서드이름

   System.out::print
   String::toUpperCase
